# Setup & Configuration Guide

This repo is your **GitHub profile README** — the special repo GitHub shows on
your profile page when the repo name exactly matches your username
(`M-Ehtasham-SE/M-Ehtasham-SE`).

## 1. Folder structure

```
M-Ehtasham-SE/                      (repo name must match your GitHub username)
├── README.md                       ← the profile itself
├── SETUP.md                        ← this file
├── .github/
│   └── workflows/
│       └── snake.yml               ← generates the animated contribution snake
└── assets/                         ← put any custom images/icons here (optional)
```

## 2. One-time setup

1. Create a repo named **exactly** `M-Ehtasham-SE` (must match your username) — if
   it doesn't exist yet, GitHub will prompt you to make it public and
   auto-add a README when you go to create it.
2. Copy `README.md`, `SETUP.md`, and the `.github/workflows/` folder into
   that repo and push to `main`.
3. That's it for the static content — GitHub renders `README.md` as your
   profile page immediately.

## 3. Enabling the animated contribution snake (optional)

The snake animation "eats" your contribution graph and is generated by
`.github/workflows/snake.yml` using the actively-maintained
[`Platane/snk`](https://github.com/Platane/snk) action.

1. Go to your repo → **Settings → Actions → General → Workflow permissions**
   and set it to **"Read and write permissions."** (This lets the workflow
   push the generated SVG to an `output` branch — no personal access token
   needed, it uses the built-in `GITHUB_TOKEN`.)
2. Go to the **Actions** tab and manually run "Generate Snake Contribution
   Animation" once (or just wait — it also runs daily and on every push).
3. After the first successful run, an `output` branch will appear containing
   the generated SVGs. The `<img>` tag already in `README.md` points at:
   `https://raw.githubusercontent.com/M-Ehtasham-SE/M-Ehtasham-SE/output/github-contribution-grid-snake-dark.svg`
   — no further action needed, it updates automatically every day.

**No secrets are required for this workflow.**

## 4. Why the rest of the README doesn't need GitHub Actions

Everything else in the README (GitHub stats, top languages, streak, activity
graph, profile views, LeetCode card, repo pin cards) is a **live-fetched
SVG**, not a committed image. Each service reads your public GitHub/LeetCode
data fresh every time someone loads your profile page, so:

- Solve a new LeetCode problem → the card updates the next time anyone views it.
- Push new commits → your stats/streak/activity graph update the next time anyone views it.

No workflow, cron job, or secret is needed for these to "stay in sync" — that's
the point of using hosted widget services instead of committing static images.

| Section | Service | Notes |
|---|---|---|
| Stats card / Top languages / Pin cards | `github-readme-stats.vercel.app` (anuraghazra) | The de-facto standard, actively maintained, ~80k GitHub stars |
| Streak stats | `streak-stats.demolab.com` (DenverCoder1) | **Not** `.herokuapp.com` — the original Heroku domain was killed when Heroku ended free hosting; this is the current maintained mirror |

| Contribution snake | `Platane/snk` GitHub Action | Actively maintained, self-hosted via your own Action (no third-party server dependency at all) |

Avoid any badge/card URL you see elsewhere ending in `.herokuapp.com` for
these particular projects — Heroku discontinued free dynos, and most of
those links are dead. If you ever see one in a tutorial, replace the domain
with the ones in the table above and keep the same query parameters.

## 6. Customizing the theme

Most cards accept a `theme=` query parameter. The README currently uses
`tokyonight` for a clean, dark, professional look. To change it, replace
`theme=tokyonight` (or `theme=tokyo-night` for the activity graph, which uses
a slightly different theme-name format) with any of: `dark`, `radical`,
`merko`, `gruvbox`, `onedark`, `cobalt`, `dracula`, `synthwave`. Keep the same
theme name across all cards for visual consistency.

## 7. Replacing usernames later

If you ever change your GitHub or LeetCode username, update these in
`README.md`:

- Every occurrence of `M-Ehtasham-SE` (GitHub username — appears in stats
  URLs, badge links, and the profile-views counter)
- Every occurrence of `M-Ehtasham` (LeetCode username — appears in the
  LeetCode card URL and badge link)
- The LinkedIn slug in the LinkedIn badge link
- The email address in the `mailto:` link

A quick way to do this locally:

```bash
sed -i 's/M-Ehtasham-SE/NEW_GITHUB_USERNAME/g' README.md
sed -i 's/M-Ehtasham/NEW_LEETCODE_USERNAME/g' README.md
```

## 9. Troubleshooting

**A stats/streak/activity card shows a broken image or "Max retries
exceeded, cannot refresh":**
This means the shared public instance hit GitHub's API rate limit (5,000
requests/hour, shared across everyone using the free public service) — it's
not specific to your data. Fix: self-host your own instance for guaranteed
uptime, especially useful right before sending your profile link to
recruiters:
1. Fork the relevant repo (e.g. `anuraghazra/github-readme-stats`).
2. Deploy your fork to your own free Vercel account (Vercel → Add New →
   Project → import your fork).
3. Optionally add your own GitHub personal access token as a `PAT_1`
   environment variable in Vercel to raise your personal rate limit.
4. Swap `github-readme-stats.vercel.app` in `README.md` for your own
   `your-fork-name.vercel.app` domain.

**LeetCode card shows 0 or stale numbers:**
LeetCode occasionally rate-limits the card service too. It self-corrects
within a few minutes; if it persists more than a day, check that your
LeetCode username in the URL is spelled exactly as it appears in your
LeetCode profile URL (case-sensitive in some edge cases).

**Snake animation image is broken:**
This only happens if the Action in Step 3 hasn't run yet, or if "Read and
write permissions" wasn't enabled in Settings → Actions. Check the Actions
tab for a red ❌ on the workflow run and read the log — it's almost always
the permissions setting.

**Typing SVG at the top doesn't render:**
`readme-typing-svg.herokuapp.com` is one of the few Heroku-era services that
is still alive (community-maintained), but if it ever goes down, swap it for
the actively maintained fork at `readme-typing-svg.demolab.com` (same query
parameters work).
